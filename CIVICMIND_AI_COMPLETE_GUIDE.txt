1. WHAT IS CIVICMIND AI?
--------------------------------------------------------------------------------

CivicMind AI is NOT a complaint portal. It is an autonomous, AI-powered civic
management platform that transforms how citizens report urban problems and how
governments respond to them.

Traditional systems: Citizen files complaint → It goes into a pile → Ignored.
CivicMind AI:        Citizen uploads photo → 8 AI agents handle everything else.

The platform uses Google Gemini 2.0 Flash to classify, deduplicate, prioritize,
route, and verify civic issues — all without human intervention. Government
authorities only see what matters, in the right order, at the right time.

Project Stack:
  Frontend  : React + Vite + Tailwind/CSS (http://localhost:5173)
  Backend   : Node.js + Express + MongoDB (http://localhost:5000)
  AI Engine : Google Gemini 2.0 Flash API
  Realtime  : Socket.IO (live updates without page refresh)
  Maps      : Leaflet.js + OpenStreetMap / Carto dark tiles

3. THE 8 AI AGENTS (HOW IT WORKS)
--------------------------------------------------------------------------------

When a citizen submits a report, 8 Gemini-powered agents activate:

AGENT 1 — Classification Agent
  What: Analyzes the uploaded photo
  Does: Determines category (Road Damage, Garbage, Water Supply, etc.)
  Does: Rates severity (Low / Medium / High / Critical)
  Does: Generates a professional issue title and description
  Done: Without citizen filling any form — just upload photo + location

AGENT 2 — Duplicate Detection Agent
  What: Scans all existing issues within 50-meter GPS radius
  Does: If a matching issue exists, merges the report (no duplicate created)
  Does: Increases upvote count on existing issue (shows higher impact)
  Does: Notifies the original reporter that their issue gained support
  Result: Clean database, no 10 reports of same pothole confusing officials

AGENT 3 — Priority Scoring Engine
  What: Assigns a Priority Score (0–100) to every issue
  Factors:
    - Severity level (Critical = higher base score)
    - Number of upvotes and verifications
    - GPS location risk factor (historical hotspot data)
    - Weather/seasonal patterns
    - Department response time history
  Result: Authorities always see the most urgent issues first

AGENT 4 — Auto-Routing Agent
  What: Reads issue category and routes to correct government department
  Examples:
    - Road Damage    → Public Works Department (PWD)
    - Water Supply   → Water Supply Department (BWSSB / Jal Board)
    - Streetlight    → Electricity / Municipal Corp
    - Garbage        → Sanitation Department
    - Sewage         → Sewage Board
  Result: Zero manual sorting by government staff

AGENT 5 — Resolution Verification Agent
  What: When authority uploads "resolved" proof photo
  Does: Compares BEFORE photo (original report) with AFTER photo (fix proof)
  Does: Uses Gemini Vision to check if issue is genuinely resolved
  Does: Flags fake/reused photos (AI detects AI-generated or reused images)
  Result: Government cannot mark issues resolved without actual fixing
  Fallback: If AI suspects fake, issue goes to Community Vote (Agent 6)

AGENT 6 — Community Confirmation Agent
  What: Activated when AI disputes a resolution
  Does: Citizens in the area get a notification to vote YES/NO (is it fixed?)
  Rules:
    - 3 YES votes = Issue auto-marked resolved
    - 3 NO votes = Authority flagged, department head notified
  Result: Citizens become the final check on government accountability

AGENT 7 — Predictive Hotspot Agent
  What: Analyzes 6 months of historical complaint data
  Does: Identifies patterns (roads with recurring damage, monsoon risk zones)
  Does: Predicts future problem areas BEFORE complaints arrive
  Output:
    - Heatmap with colored circles (Red=Critical, Orange=High, Yellow=Medium, Green=Low)
    - "Road C — 87% pothole probability in next 30 days"
    - "Increase garbage collection near Market every Sunday"
  Result: Government can ACT before citizens suffer

AGENT 8 — Civic Chat Assistant
  What: Natural language AI chatbot embedded in the platform
  Does: Answers questions using real database statistics
  Does: Falls back to rule-based smart answers when Gemini quota is low
  Examples:
    - "How many issues are unresolved?" → Real DB count
    - "Which area has most garbage complaints?" → Top category from DB
    - "How do I earn points?" → Explains reward system
  Result: Citizens get instant help without contacting government

5. HOW TO RUN THE PROJECT
--------------------------------------------------------------------------------

Prerequisites:
  - Node.js v18+
  - MongoDB running locally (or MongoDB Atlas URI)
  - Google Gemini API Key

Step 1 — Backend:
  cd CivicMindAI/backend
  npm install
  Create .env file with:
    PORT=5000
    MONGODB_URI=mongodb://localhost:27017/civicmind
    JWT_SECRET=your_secret_key_here
    GEMINI_API_KEY=your_gemini_api_key_here
  node server.js
  → Backend runs on http://localhost:5000

Step 2 — Frontend:
  cd CivicMindAI/frontend
  npm install
  npm run dev
  → Frontend runs on http://localhost:5173

Step 3 — Seed Demo Data (optional):
  cd CivicMindAI/backend
  node seeders/seed.js
  → Creates demo users + sample issues for testing

7. ARCHITECTURE
--------------------------------------------------------------------------------

FRONTEND PAGES:
  /              → Landing.jsx        (public, always dark theme)
  /dashboard     → CitizenDashboard.jsx / AuthorityDashboard.jsx
  /report        → ReportIssue.jsx    (3-step wizard)
  /map           → MapView.jsx        (Leaflet map)
  /community     → CommunityFeed.jsx  (feed + detail modal)
  /chat          → ChatAI.jsx         (chat interface)
  /leaderboard   → Leaderboard.jsx    (points + badges)
  /predictions   → PredictiveAnalytics.jsx (6-tab analytics)
  /profile       → Profile.jsx        (user settings)

BACKEND API ROUTES:
  POST /api/auth/register    → Register new user
  POST /api/auth/login       → Login, returns JWT token
  GET  /api/auth/me          → Get current user
  PUT  /api/auth/profile     → Update profile details

  GET  /api/issues           → List issues (with filters)
  POST /api/issues           → Create new issue (triggers AI)
  PUT  /api/issues/:id/status    → Update status (authority)
  POST /api/issues/:id/verify    → Community verify
  POST /api/issues/:id/upvote    → Upvote
  POST /api/issues/:id/resolve   → Authority resolution + AI verify
  POST /api/issues/:id/confirm   → Community confirm resolution

  POST /api/ai/chat          → AI chat message
  GET  /api/ai/predictions   → Get AI predictions + analytics

  GET  /api/notifications    → Get user notifications
  PUT  /api/notifications/read → Mark all read

  GET  /api/leaderboard      → Top contributors

BACKEND AI FILES:
  ai/classificationAgent.js  → Photo → category, severity, title, description
  ai/duplicateAgent.js       → Nearby issue check
  ai/priorityAgent.js        → Priority score calculation
  ai/routingAgent.js         → Category → Department mapping
  ai/resolutionAgent.js      → Before/after photo verification
  ai/predictiveAgent.js      → Hotspot + trend analysis
  ai/chatAgent.js            → Civic chat with DB fallback

DATABASE MODELS:
  User.js    → name, email, password, role, points, badges, profile fields
  Issue.js   → title, description, category, severity, status, location,
               images, priorityScore, upvotes, verifications, assignedDept,
               resolutionVerification, communityConfirms, duplicateCount
  Notification.js → user, title, message, isRead, createdAt

SOCKET EVENTS:
  connection           → User joins, enters area room
  join-area           → Subscribe to location-based updates
  new-issue           → Broadcast when new issue submitted
  issue-update        → Broadcast when status changes
  notification        → Push notification to specific user

9. KEY DECISIONS & WHY
--------------------------------------------------------------------------------

Q: Why does the landing page not change with light/dark mode?
A: The landing page is a marketing page with dark atmospheric design.
   Using theme variables made text invisible in light mode.
   Fix: All landing sections use hardcoded dark colors (rgba, white text).

Q: Why does the map legend always have dark background?
A: Leaflet maps render on white/light tiles. The legend panel was using
   CSS variables that become dark text on dark background in dark mode.
   Fix: map-legend class hardcoded to rgba(8,4,24,0.92) + white text.

Q: Why does chat still work even when Gemini API quota is exceeded?
A: chatAgent.js has a full smart fallback:
   - Tries Gemini first
   - On error (429 quota), switches to keyword-matching DB queries
   - 12+ question types handled: stats, rewards, reporting, hotspots, etc.
   - Always returns a useful response, never an error message

Q: Why are predictive hotspots visible even with no data?
A: DEMO_HOTSPOTS array in PredictiveAnalytics.jsx provides 6 pre-defined
   Delhi NCR risk zones when Gemini API returns empty predictions.
   Map auto-centers to Delhi with these demo circles.

Q: Can authorities raise issues?
A: No — by design. Authorities only resolve issues raised by citizens.
   This ensures AI duplicate detection and priority scoring isn't bypassed.
   The AI pipeline only works correctly when issues come from citizens
   with location data and photos.

Q: What happens if someone submits a fake resolution photo?
A: Gemini Vision compares the before and after photos:
   - If the issue looks genuinely fixed: marked Resolved
   - If photo looks reused/fake/unrelated: status = "Needs Further Action"
   - Community voting activated: citizens in the area vote YES/NO
   - 3 NO votes = Authority department flagged to their supervisor
